Лабораторная работа: Графовые базы данных в MS SQL Server
Вариант 65: Архитектура города — Памятники, Улицы, Архитекторы

Условие задачи:
Разработать графовую базу данных в MS SQL Server на тему "Архитектура города",
включающую сведения о памятниках архитектуры, улицах и архитекторах Москвы.

Узлы (NODE таблицы):
1. Architects (Архитекторы) — имя, годы жизни, национальность, специализация
2. Monuments (Памятники) — название, год постройки, стиль, высота, описание
3. Streets (Улицы) — название, длина, район, ширина, год основания

Рёбра (EDGE таблицы):
1. Located_On — памятник РАСПОЛОЖЕН НА улице
   Атрибуты: дата размещения, описание сектора, признак главного входа
   CONNECTION: Monuments TO Streets

2. Designed_By — объект СПРОЕКТИРОВАН архитектором
   Атрибуты: год проектирования, роль, полученная награда
   CONNECTION: Monuments TO Architects, Streets TO Architects

3. Collaborated_With — архитектор СОТРУДНИЧАЛ С другим архитектором
   Атрибуты: название проекта, год сотрудничества, тип сотрудничества
   CONNECTION: Architects TO Architects

4. Connects_To — улица СОЕДИНЯЕТСЯ С другой улицей
   Атрибуты: тип пересечения, уровень трафика, год строительства
   CONNECTION: Streets TO Streets

Примеры запросов MATCH:
- Все памятники и их архитекторы
- Все памятники и улицы их расположения
- Памятник → улица → смежная улица (цепочка 3 узла)
- Памятники стиля "Сталинский ампир" → архитектор → его соавтор (цепочка 3 узла)
- Памятник → улица → смежная улица → архитектор смежной улицы (цепочка 4 узла)
- Соавторы архитекторов и все памятники первого автора

Примеры запросов SHORTEST_PATH:
- Кратчайший путь от Тверской улицы до всех достижимых улиц (шаблон +)
- Кратчайший путь через сотрудничество от Щусева (шаблон {1,4})
